# 피보나치 수열: 순환적 방법 vs 재귀적 방법 성능 비교

## 개요

피보나치 수열을 **순환적(반복적, Iterative)** 방법과 **재귀적(Recursive)** 방법으로 각각 구현하고,  
정수 N을 증가시켜가며 두 방법의 수행 시간을 프로파일링한다.

> 피보나치 수열 정의: F(1) = 1, F(2) = 1, F(n) = F(n-1) + F(n-2) (n ≥ 3)

---

## 구현

### 순환적 방법 (`fibonacci_iterative.c`)

```c
long long fibonacci_iterative(int n) {
    if (n == 1 || n == 2) return 1;

    long long prev2 = 1, prev1 = 1, curr = 0;
    for (int i = 3; i <= n; i++) {
        curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return curr;
}
```

- 루프를 이용해 이전 두 값만 유지하며 순차적으로 계산한다.
- 시간 복잡도: **O(n)**
- 공간 복잡도: **O(1)**

---

### 재귀적 방법 (`fibonacci_recursive.c`)

```c
long long fibonacci_recursive(int n) {
    if (n == 1 || n == 2) return 1;
    return fibonacci_recursive(n - 1) + fibonacci_recursive(n - 2);
}
```

- 정의를 그대로 코드로 표현한다.
- 시간 복잡도: **O(2ⁿ)** — 중복 호출이 기하급수적으로 증가
- 공간 복잡도: **O(n)** — 재귀 호출 스택 깊이

## 비교 그래프

아래는 두 방법의 수행 시간을 시각적으로 비교한 것이다 (공통 측정 지점 기준).

```
수행시간 (초, 로그 스케일)
10^1  |                                              *  [재귀 N=45]
10^0  |
10^-1 |                                   *             [재귀 N=40]
10^-2 |
10^-3 |                        *                        [재귀 N=30]
10^-4 |             *                                   [재귀 N=20]
10^-5 |   * *  *                                        [재귀 N=1~10]
10^-6 |---*--*--*--*--*--*--*--*--*--*--*-------------- [순환 N=1~90]
       N=1  5  10  20  30  40  45  50 ...
```

- **순환적 방법**: 로그 스케일에서 거의 수평 (일정 시간)
- **재귀적 방법**: N이 증가할수록 지수적으로 급증

## 분석 및 결론

### 시간 복잡도 비교

| 항목           | 순환적 방법 | 재귀적 방법 |
|----------------|-------------|-------------|
| 시간 복잡도    | O(n)        | O(2ⁿ)       |
| 공간 복잡도    | O(1)        | O(n)        |
| N=40 수행시간  | ~0.000000 초 | ~0.685 초  |
| N=45 수행시간  | ~0.000000 초 | ~6.84 초   |
| 실용적 N 범위  | 매우 큰 값도 가능 | 약 N≤45 한계 |

### 재귀적 방법의 문제점: 중복 계산

재귀 방법은 동일한 부분 문제를 반복해서 계산한다.  
예를 들어 `F(5)`를 구할 때:

```
F(5)
├── F(4)
│   ├── F(3)
│   │   ├── F(2) ← 중복
│   │   └── F(1) ← 중복
│   └── F(2) ← 중복
└── F(3)
    ├── F(2) ← 중복
    └── F(1) ← 중복
```

`F(2)`가 3번, `F(3)`이 2번 계산된다. N이 커질수록 중복 횟수는 **지수적으로** 증가한다.

### 재귀적 방법의 장점

- 코드가 수학적 정의와 1:1 대응되어 **가독성이 높다**.
- 구현이 단순하여 **이해하기 쉽다**.
- 메모이제이션(memoization)을 추가하면 O(n)으로 개선 가능하다.

### 결론

| 상황 | 권장 방법 |
|------|-----------|
| 실제 계산이 필요한 경우 | 순환적 방법 |
| 알고리즘 학습 / 이해 | 재귀적 방법 |
| 성능이 중요한 경우 | 순환적 방법 또는 메모이제이션 재귀 |

순환적 방법이 **시간 효율성** 면에서 압도적으로 우수하다.  
재귀적 방법은 O(2ⁿ)의 시간 복잡도로 인해 N이 50을 넘으면 실용적으로 사용하기 어렵다.  
그러나 재귀적 표현은 개념의 명확성 면에서 가치가 있으며, 동적 프로그래밍(메모이제이션)과 결합하면 성능 문제를 해결할 수 있다.
